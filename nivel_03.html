Construindo em Camadas: Desafio de Programação Orientada a Objetos (OOP) e Docker

Mergulhamos no mundo sofisticado da Programação Orientada a Objetos (OOP) e das poderosas tecnologias de contêiner com Docker. Estes desafios é uma jornada emocionante e desafiadora para os desenvolvedores que desejam elevar suas habilidades para o próximo nível. Você aprenderá a conceber e implementar sistemas complexos com a OOP, criando código modular, reutilizável e facilmente mantido. Além disso, mergulharemos nas maravilhas do Docker, onde você ganhará habilidades essenciais para empacotar suas aplicações em ambientes isolados, tornando o processo de desenvolvimento, distribuição e implantação mais eficiente do que nunca. Estes desafios não apenas expandirá seu conhecimento técnico, mas também abrirá portas para uma abordagem mais avançada e profissional no desenvolvimento de software.

Lista de exercícios


Exercício 01

Antes de iniciar o desafio, leia a referência, para saber mais informações:

  1. Classes e Objetos no PHP

Crie uma classe PHP de Produto, adicione dentro dessa classe as seguintes propriedades: nome, descricao, preco. Crie 2 objetos instanciando a classe Produto e imprima o nome dos dois objetos.


Exercício 02

Antes de iniciar o desafio, leia a referência, para saber mais informações:

  1. constructor no PHP

Pegue como continuacao a classe do exercicio 01, adicione um construtor para a classe Produto. Esse construtor deve receber os valores nome, descricao e preco no momento que instanciar para um novo objeto. Além disso, ajuste as propriedades para que elas não fiquem publicas, devem ficar como privadas.


Exercício 03

Pegue como continuacao a classe do exercicio 02, crie os métodos Getters e Setters das propriedades nome, descricao e preco. No setter do nome certifique-se que sempre a primeira letra seja maiúscula.


Exercício 04

Em dupla crie um github adicionando o seu parceiro como membro. Os dois em suas máquinas devem clonar o repositório, realizar modificações e subir no repositório essas modificações. Os dois devem dar git pull dessas modificações para terem os mesmos códigos em suas máquinas.
